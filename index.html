    Bayesian Logistic Regression¶

    I am first going to explore concepts in logistic regression. Then I'm going to add a bayesian twist to this ML algorithm and apply it to a couple of familiar problems. Logistic regression takes its name from a logistic equation being applied as a thresholding function, in this case $$z=\frac{1}{(1+exp^{-y})}$$ to the familiar regression line $$y = \alpha + \beta X$$.

    Contrary to what one might think at first hearing, this is an algorithm suitable for a classification problem; not a regression problem.

    I start first with a simple classification problem where the outcome has to be one of two possible classes labeled 0 and 1. Because of the binary outcome, and remembering Kruschke's puppy book, I use the Bernoulli distribution. In reference to the puppy book, the parameterization can be thought of as $$\theta=expit(\alpha + \beta X)$$ and the label $$l \sim Bern\left(\theta\right)$$ where $$l \in [0,1]$$

    My Goal:¶

    In most data science tutorials I have seen, a lot of the data clean-up is done in what seems to me casually, an annoying obstacle to get to the sexy Machine Learning bit. I was curious to see what if any difference it made in my Kaggle ranking if I used a somewhat more cautious approach in my data cleanup. My approach was to dumbly follow Datacamp's tutorial and submit my test set labels as a benchmark. The second step is then to use a fancier data cleanup process and see whether taking the extra time actullay moves me up in the ranking (or perhaps down?).

    Imputing Age:¶

    Age is one of those things that to a first approximation can be estimated by how a person is referred to. First I am going to catalogue titles present in names and create another feature, "Title". To do this I need to do an inventory of all possible titles present in the dataset. Titles seem to be the seconde word in the name string, ending with a '.'.

    In [23]:

        # Now I grab the part that has a '.' after but verify it is an abbreviation, not an initial...

            cnt += 1
            nameset.add(subname)
